# セッション引き継ぎファイル生成

次のセッションで作業を再開できるよう、現在の状態を `.claude/handoff/session-{timestamp}.md` に書き出します。

## 手順

### Step 1: 情報収集

以下のコマンドを **並行** で実行して情報を収集する：

```powershell
# 現在のブランチ
git -C $projectRoot branch --show-current

# 変更ファイル一覧
git -C $projectRoot status --short

# 直近 15 コミット
git -C $projectRoot log --oneline -15
```

`$projectRoot` は現在の作業ディレクトリ（プロジェクトルート）を指す。

### Step 2: 会話コンテキストの整理

**現在の会話から以下をまとめる**（ツールで取得するのではなく、この会話の記憶から作成する）：

1. **主なリクエスト・目的**: ユーザーが取り組んでいるタスクの概要
2. **技術的コンテキスト**: 重要な設計決定・発見・制約事項
3. **変更ファイル詳細**: 変更した各ファイルと変更内容の要点
4. **エラーと対処**: 発生したエラーとその解決方法
5. **未解決タスク**: まだ完了していないタスクと現在の状況
6. **直近の会話フロー**: ユーザーとのやり取りの流れ（最終 10〜15 件）

### Step 3: メモリインデックスの読み込み

`{projectRoot}/memory/MEMORY.md` が存在すれば Read ツールで読み込む。存在しない場合はスキップ。

### Step 4: 引き継ぎファイルの書き出し

**タイムスタンプ**: `yyyy-MM-dd-HH-mm` 形式（例: `2026-05-07-14-30`）

**出力先**: `{projectRoot}/.claude/handoff/session-{timestamp}.md`

**ファイル構造**:

```markdown
# セッション引き継ぎファイル

**生成日時**: {datetime}
**生成方法**: /handoff スキル（手動）
**プロジェクトルート**: {projectRoot}
**追加メモ**: {$ARGUMENTS または "なし"}

---

## Git 状態

### 現在のブランチ
```

{gitBranch}

```

### 変更ファイル (git status --short)
```

{gitStatus}

```

### 直近 15 コミット
```

{gitLog}

```

---

## 主なリクエストと目的

{現在のセッションでユーザーが取り組んでいたことの概要}

---

## 技術的コンテキスト

{重要な設計決定・発見・制約事項を箇条書きで}

---

## 変更ファイル詳細

{変更した各ファイルと変更内容の要点}

---

## エラーと対処

{発生したエラーと解決方法（未解決のものも含む）}

---

## 未解決タスク

{完了していないタスクと現在の状況}

---

## メモリインデックス (MEMORY.md)

{MEMORY.md の内容、または "（なし）"}

---

## 直近の会話フロー

{ユーザーとのやり取りの要点（最終 10〜15 件、各発言を簡潔に）}

---

*このファイルは /handoff スキルにより手動生成されました。*
*次のセッション開始時にこのファイルを参照して作業文脈を復元してください。*
```

### Step 5: 完了報告

ファイルを書き出したら、以下を出力する：

```
引き継ぎファイルを生成しました:
  {handoffFilePath}

次のセッション開始時にこのファイルをコンテキストとして提供してください。
```

## 注意事項

- 引き継ぎファイルは「次のセッションで新しい Claude が読む」ことを前提に書く
- コードの全文は含めず、**変更の意図・制約・未解決の課題** を重点的に記録する
- エラーや失敗した試みも積極的に記録する（同じ失敗を繰り返さないため）
- `$ARGUMENTS` に追加メモが指定された場合は「追加メモ」セクションに含める
