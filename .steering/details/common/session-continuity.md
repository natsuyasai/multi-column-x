# セッション継続テンプレート

## ウェルカムバックプロンプトテンプレート

ユーザーが既存のAI-DLCプロジェクトの作業を継続するために戻ってきた際に、このプロンプトを表示します：

```markdown
**お帰りなさい！進行中のAI-DLCプロジェクトが見つかりました。**

aidlc-state.mdに基づく現在の状況：
- **プロジェクト**：[プロジェクト名]
- **現在のフェーズ**：[INCEPTION/CONSTRUCTION/OPERATIONS]
- **現在のステージ**：[ステージ名]
- **最後に完了した内容**：[最後に完了したステップ]
- **次のステップ**：[次に取り組む内容]

**今日は何に取り組みますか？**

A) 中断した箇所から続ける（[次のステップの説明]）
B) 以前のステージをレビューする（[利用可能なステージを表示]）

[Answer]:
```

## 必須：セッション継続の手順

1. **既存プロジェクトを検出した場合は必ずaidlc-state.mdを最初に読む**
2. **ワークフローファイルから現在の状態を解析**してプロンプトに入力する
3. **必須：以前のステージ成果物を読み込む** - ステージを再開する前に、以前のステージから関連する成果物をすべて自動的に読み込む：
   - **リバースエンジニアリング**：architecture.md、code-structure.md、api-documentation.mdを読み込む
   - **要件分析**：requirements.md、requirement-verification-questions.mdを読み込む
   - **ユーザーストーリー**：stories.md、personas.md、story-generation-plan.mdを読み込む
   - **アプリケーション設計**：アプリケーション設計成果物（components.md、component-methods.md、services.md）を読み込む
   - **設計（ユニット）**：unit-of-work.md、unit-of-work-dependency.md、unit-of-work-story-map.mdを読み込む
   - **ユニットごとの設計**：functional-design.md、nfr-requirements.md、nfr-design.md、infrastructure-design.mdを読み込む
   - **コードステージ**：すべてのコードファイル、計画、および以前のすべての成果物を読み込む
4. **ステージ別のスマートコンテキスト読み込み**：
   - **初期ステージ（ワークスペース検出、リバースエンジニアリング）**：ワークスペース分析を読み込む
   - **要件/ストーリー**：リバースエンジニアリング + 要件成果物を読み込む
   - **設計ステージ**：要件 + ストーリー + アーキテクチャ + 設計成果物を読み込む
   - **コードステージ**：すべての成果物 + 既存のコードファイルを読み込む
5. **アーキテクチャの選択と現在のフェーズに基づいてオプションを適応**させる
6. **汎用的な説明ではなく具体的な次のステップを表示**する
7. **継続プロンプトをタイムスタンプ付きでaudit.mdにログ記録**する
8. **コンテキストサマリー**：成果物の読み込み後、ユーザーに何が読み込まれたかの簡単なサマリーを提供する
9. **質問する場合**：明確化やユーザーフィードバックの質問は必ず.mdファイルに記載する。チャットセッションの本文に複数選択式の質問を直接記載しないこと。

## エラー処理

セッション再開時に成果物が見つからないまたは破損している場合は、回復手順についての指針を[error-handling.md](error-handling.md)で参照してください。
