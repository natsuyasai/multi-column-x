# 👋 AI-DLC（AIドリブン開発ライフサイクル）へようこそ！ 👋

あなたの特定のニーズに合わせて自己適応するアダプティブなソフトウェア開発ワークフローをご案内します。

## AI-DLCとは？

AI-DLCは、プロジェクトのニーズに適応する構造化された柔軟なソフトウェア開発プロセスです。経験豊富なソフトウェアアーキテクトとして以下のことを行います：

- **要件を分析**し、必要に応じて明確化の質問をする
- **最適なアプローチを計画**し、複雑さとリスクに基づいて判断する
- **シンプルな変更では不必要なステップをスキップ**し、複雑なプロジェクトでは包括的なカバレッジを提供する
- **すべてを文書化**して、決定事項と根拠の完全な記録を残す
- **各フェーズを案内**し、明確なチェックポイントと承認ゲートを提供する

### フェーズの詳細

**INCEPTION PHASE** - *計画とアプリケーション設計*

- **目的**：何を作るか（WHAT）と理由（WHY）を決定する
- **活動**：要件の理解、既存コードの分析（ある場合）、アプローチの計画
- **成果物**：明確な要件、実行計画、並行開発のための作業単位数の決定
- **あなたの役割**：質問に答え、計画をレビューし、方向性を承認する

**CONSTRUCTION PHASE** - *詳細設計、実装、テスト*

- **目的**：どのように作るか（HOW）を決定する
- **活動**：詳細設計（必要な場合）、コード生成、包括的なテスト
- **成果物**：動作するコード、テスト、ビルド手順
- **あなたの役割**：設計をレビューし、実装計画を承認し、結果を検証する

**OPERATIONS PHASE** - *デプロイとモニタリング（将来）*

- **目的**：どのようにデプロイして運用するか（DEPLOY and RUN）
- **ステータス**：将来のデプロイとモニタリングワークフローのプレースホルダー
- **現在の状態**：ビルドとテストの活動はCONSTRUCTIONフェーズで処理される

## 主要原則

- ⚡ **完全適応型**：各ステージはあなたのニーズに基づいて独立して評価される
- 🎯 **効率的**：シンプルな変更は必須ステージのみを実行
- 📋 **包括的**：複雑な変更はすべての保護措置を含む完全な処理を受ける
- 🔍 **透明性**：作業開始前に実行計画を確認・承認できる
- 📝 **文書化**：すべての決定と変更の完全な監査証跡
- 🎛️ **ユーザーコントロール**：ステージの追加や除外をリクエストできる

## 次に行われること

1. **ワークスペースを分析**して、新規または既存のプロジェクトかを判断
2. **要件を収集**し、必要に応じて明確化の質問をする
3. **実行計画を作成**し、実行を提案するステージとその理由を示す
4. **計画をレビューして承認**（または変更をリクエスト）
5. **計画を実行**し、主要なステージごとにチェックポイントを設置
6. **完全なドキュメントとテストを含む動作するコード**を提供

AI-DLCプロセスは以下に適応します：

- 📋 あなたの意図の明確さと複雑さ
- 🔍 既存のコードベースの状態
- 🎯 変更のスコープと影響
- ⚡ リスクと品質の要件

それでは始めましょう！
